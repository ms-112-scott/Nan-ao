# 南澳知識圖譜計畫 — 詳細規劃 v1

> 參考：[ms-112-scott/Zhudong](https://github.com/ms-112-scott/Zhudong)
> 主題：宜蘭南澳（泰雅部落 Klesan 群 + 漢人移墾 + 林業/古道/海岸聚落）
> 形式：**單頁應用（SPA）**，所有功能集中在一頁
> 階層：3 層（分類 → 子類 → 節點）
> 視覺基調：文史溫潤紙質感（米色底 + 襯線字 + 手繪線條）
> 突破點欄位：保留 + 特殊視覺
> 規模：節點 100–1000、階層 + 跨越混合關係、有時間維度但無分期

## 0. 與竹東計畫的關係：什麼沿用、什麼換掉

| 面向 | 竹東 | 南澳（本計畫） | 理由 |
|---|---|---|---|
| 應用形式 | 多頁網站（Home/Gallery/Workshop/Education/KnowledgeGraph） | **單頁集中** | 你明確要求；也降低維護負擔 |
| 主題範疇 | 客庄人物、產業、地景 | 泰雅部落 + 漢人移墾 + 林業海防（雙文化結構） | 南澳的歷史本質就是混雜的 |
| 視覺基調 | 暗色科技感（#0a0a0c 深底 + 螢光節點） | **米色紙質感 + 襯線字 + 手繪線條** | 與文史檔案的閱讀直覺一致 |
| 階層 | 平面（無 parent_id） | **3 層樹 + 跨越邊** | 100–1000 規模一定要有摺疊 |
| 資料管線 | Google Sheets + Apps Script（即時拉） | **本地 Excel → GitHub Action → JSON**（建議） | 詳見 §12，三選一 |
| 時間維度 | start_year / end_year + 滑桿 | 同 + **跨年事件以橫條呈現** | 沿用 |
| 渲染 | D3 force-directed（在 SVG/Canvas 之間） | **D3 + Canvas（強制）** | 1000 節點 SVG 一定卡 |
| 突破點 | 無此欄位 | **新增 breakthrough_note + 金色光暈視覺** | 你的研究需求 |

沿用：力導向的物理參數、InfoCard 右側 slide-in、搜尋聚焦飛行、時間滑桿、圖例可點即篩。

換掉：暗色配色全部改、SVG 路徑要改成 Canvas、節點/邊的繪製器要重寫一套紙質感版本、加入階層摺疊邏輯、新增 minimap、頂層「分類/子類/節點」三段式樹狀 sidebar（左側）。

## 1. 南澳主題的資料模型特殊考量

南澳的資料天生有「雙文化結構」，schema 必須容得下：

**A. 泰雅文化軸**
- 部落（武塔、東岳、南澳、碧候、金洋、澳花）為地理錨點
- 家系/gaga/ssan 等族群結構（可能出現節點如「某某 ssan」）
- 口傳故事、神話地景（與「事件」不同，沒有明確年份）
- 遷徙路線（從南投發祥地 → 大濁水溪流域 → 現址）

**B. 漢人移墾與國家治理軸**
- 清代開山撫番、日治蕃地、戰後伐林、林務局接管
- 武塔、碧候、金洋為集團移住成果
- 古道（南澳古道、比亞毫、舊武塔）

**C. 自然地景軸**
- 南澳南北溪、澳花瀑布、神祕海灘、烏石鼻、烏帽山
- 與部落/事件強耦合（地景不是裝飾，是節點）

→ 這意味著節點的 `category`（分類）需要至少 5–7 大類，比竹東的單文化框架更寬。

**建議分類（第 1 層）：**
1. 部落聚落（pulan / qalang）
2. 人物（族人、漢人、官員、學者）
3. 事件（遷徙、行政、衝突、開發）
4. 地景（自然、古道、遺址）
5. 物質文化（口傳、儀式、工藝、語言）
6. 機構與組織（部落會議、教會、林管處、合作社）
7. 文獻與媒體（古地圖、調查報告、紀錄片）

→ 子類（第 2 層）由文史工作者填，例如「事件 → 集團移住」、「人物 → 部落耆老」。

→ 節點（第 3 層）是實際資料點。

## 2. 技術架構選型（最終）

### 2.1 框架
- **React 19 + Vite**（同 Zhudong，重用工程習慣）
- **不用** react-router-dom（單頁，不需路由切換頁面）
- **Tailwind v4**（同 Zhudong，但配色 token 全換）

### 2.2 渲染
- **D3 v7 + Canvas**（強制，SVG 在 500+ 節點會卡）
- 節點命中測試：`d3-quadtree`
- 文字 label：DOM overlay（絕對定位 div），而不是 Canvas 描繪——因為紙質感要用襯線字、要支援字符精細排版，Canvas 文字渲染品質差

→ 這是與 Zhudong `ForceGraph.js` 最大的內部差異。竹東的 `drawNodes`/`drawLinks` 用 D3 selection 操作 SVG-like API；本計畫要重寫成 Canvas 版本，並另外維護一個 DOM label 圖層（只渲染視窗內的節點 label，避免 1000 個 div）。

### 2.3 資料管線（三選一，建議 B）

**A. 沿用 Zhudong 的 Google Sheets + Apps Script**
- 優點：文史工作者直接編 Sheet，網站即時更新；驗證會把錯誤儲存格塗紅/塗黃
- 缺點：Apps Script 有配額；網站每次載入要打 API；無版本歷史
- 適合：頻繁編輯、多人協作、現在這個階段

**B. 本地 Excel → GitHub Action → JSON（推薦）**
- 流程：文史工作者編 Excel → 推到 repo `data/source.xlsx` → GitHub Action 跑轉換腳本 → 產出 `data/data.json` → commit → Pages deploy
- 優點：JSON 跟程式碼一起進版控，可回溯；網站載入快；驗證在 CI 跑，PR 時就能看到錯誤
- 缺點：文史工作者要會 git push（或請人代推）
- 適合：有穩定的內容貢獻者、希望資料品質可審核

**C. 純 JSON 手刻**
- 適合：你自己既是研究者又是開發者、節點數量在 200 以下

→ 建議 B。可以先 A 試水溫，等資料穩定後切換 B。

### 2.4 託管
- GitHub Pages，路徑 `https://<user>.github.io/Nanao/`
- 自訂網域可後加（DNS CNAME → `<user>.github.io`）

### 2.5 狀態管理
- **URL query string 為單一真實來源**
- 參數：`?node=<id>&view=graph|timeline&filter=cat1,cat2&years=1900-2025&edges=hierarchy|cross|both&collapsed=g1,g2`
- 用 `URLSearchParams` + `popstate` 監聽；不用 react-router

## 4. 資料 Schema（細化版）

### 4.1 nodes 表

| 欄位 | 型別 | 必填 | 說明 |
|---|---|---|---|
| `id` | string | ✓ | 全域唯一，建議用「類別前綴_中文短碼」如 `event_集團移住1916`、`person_莎韻`、`place_武塔` |
| `title` | string | ✓ | 顯示名稱 |
| `category` | enum | ✓ | 第 1 層分類，從 §1 七大類選 |
| `subcategory` | string |  | 第 2 層子類，自由文字（之後會聚合成 enum） |
| `parent_id` | string |  | 階層父節點 id（空值表示頂層） |
| `year` | int |  | 起始年（西元，泰雅敘事可用「約 1850」→ 1850） |
| `year_end` | int |  | 結束年（持續事件用，單點事件留空） |
| `year_uncertain` | bool |  | 年份是口傳推估（視覺化會加問號標記） |
| `summary` | string | ✓ | 一句話摘要（≤ 50 字，圖上 hover 顯示） |
| `long_text` | markdown |  | 詳細介紹（InfoCard 顯示，支援 markdown） |
| `breakthrough_note` | markdown |  | **突破點專屬**：新發現/翻案/未解之謎，視覺上加金色光暈 |
| `images` | string |  | 檔名列表，分號分隔（如 `wuta_1923.jpg;wuta_now.jpg`） |
| `youtube_urls` | string |  | YouTube URL 列表，分號分隔 |
| `audio_urls` | string |  | 口傳/訪談音檔 URL（為泰雅口述新增） |
| `sources` | markdown |  | 出處（書籍頁碼、訪談日期、檔案編號） |
| `importance` | int 1-5 | ✓ | 視覺大小映射；3 為預設 |
| `tags` | string |  | 自由標籤，分號分隔（搜尋用） |
| `coords` | string |  | 經緯度 `lon,lat`（為地圖視圖預留，Phase 2） |
| `clan` | string |  | 泰雅家系/ssan 名稱（南澳特化欄位） |
| `language` | enum |  | 主要記載語言：`zh`/`tay`（泰雅羅馬字）/`ja`/`mixed` |

### 4.2 edges 表

| 欄位 | 型別 | 必填 | 說明 |
|---|---|---|---|
| `source_id` | string | ✓ | 起點節點 id |
| `target_id` | string | ✓ | 終點節點 id |
| `edge_type` | enum | ✓ | `hierarchy`（階層）/ `cross`（跨越）/ `temporal`（時序因果） |
| `relation_label` | string |  | 顯示文字（如「遷徙至」「師承」「衍生」） |
| `directed` | bool |  | 是否有方向；預設依 edge_type（hierarchy=true） |
| `weight` | int 1-5 |  | 線粗映射；預設 2 |
| `note` | string |  | 關係佐證/出處 |

### 4.3 metadata 表（可選）

| 欄位 | 說明 |
|---|---|
| `category_colors` | 各 category 對應的紙質感色票（見 §5） |
| `last_updated` | 自動填入轉換時間 |
| `data_version` | 對應 git commit hash |

### 4.4 驗證規則（GitHub Action 跑）

1. `id` 全域唯一，違反 → 紅
2. `parent_id` 必須指向存在的 node id
3. `category` 必須在 enum 中
4. `edges.source_id`/`target_id` 必須指向存在的 node id（缺漏 → 黃）
5. `year_end >= year`（如果都有填）
6. `images` 列出的檔案必須真的存在於 `media/img/`
7. `breakthrough_note` 不可超過 500 字（強制簡潔）
8. 偵測「孤兒節點」：沒有任何 edge 連入連出 → 警告（不擋 build）

## 5. 視覺語言：紙質感詳細規格

### 5.1 配色（Light Mode 為主，紙質感不適合 Dark Mode）

```
基底
--paper-bg:        #F4EEE0   /* 淡米色，主背景 */
--paper-bg-deep:   #E8DFCC   /* 加深米色，面板/卡片底 */
--paper-edge:      #D6C9B0   /* 紙邊陰影色 */
--ink-primary:     #2A2620   /* 墨色，主要文字 */
--ink-secondary:   #5C5448   /* 次級文字 */
--ink-faint:       #9C9180   /* 弱文字、線條 */

節點分類色（飽和度低、有泥土感）
--cat-tribe:       #8B5A2B   /* 部落聚落 — 赭石褐 */
--cat-person:      #4A6B5B   /* 人物 — 苔綠 */
--cat-event:       #A8412B   /* 事件 — 朱紅 */
--cat-landscape:   #5E7B8C   /* 地景 — 山藍灰 */
--cat-culture:     #9B7B3F   /* 物質文化 — 黃褐 */
--cat-org:         #6B5B7B   /* 機構 — 暗紫 */
--cat-doc:         #7B6B5B   /* 文獻 — 牛皮褐 */

突破點專屬
--breakthrough:    #C9A227   /* 沉穩金 */
--breakthrough-glow: rgba(201,162,39,0.35)
```

### 5.2 字體
- 中文標題：**思源宋體 Source Han Serif TC**（Heavy / Bold）
- 中文內文：**思源宋體**（Regular）
- 中文副標：**思源黑體**（Light） — 提升可讀性對比
- 數字 / 年份：**EB Garamond** 或 **Cardo**（西文襯線）
- 自託管在 `public/fonts/`，避免 Google Fonts CDN 被牆/失敗

### 5.3 節點造型
- 形狀：**圓形**為主，但 7 大類各微調筆觸
  - 部落：實心圓 + 外圈虛線（似印章）
  - 人物：實心圓 + 細白邊
  - 事件：六邊形（事件感）
  - 地景：圓角矩形（似地標牌）
  - 文化：菱形（儀式感）
  - 機構：方形 + 內陰影
  - 文獻：書頁形（直立矩形 + 一角摺角）
- 大小：直徑 = `8 + importance * 4` px（importance 1=12px, 5=28px）
- 突破點節點：在原形狀外加 1.5x 半徑的金色徑向漸層光暈 + 8 角小星標於右上

### 5.4 邊樣式
- **階層邊（hierarchy）**：粗實線 1.5px、墨色 `--ink-primary` 60% 透明、有箭頭（指向子節點）
- **跨越邊（cross）**：細手繪感虛線 0.8px、墨色 50% 透明、無箭頭、輕微貝茲曲線（避免直線交織）
- **時序因果邊（temporal）**：細點線 0.8px、朱紅色 40% 透明、有箭頭、曲度比跨越更大
- 手繪感達成方式：每段線分成 3–5 個微抖動控制點（offset ±0.5px），不是真的 SVG filter，是 Canvas 繪製時加 perlin noise

### 5.5 紙質感背景
- 主背景貼一張 1024×1024 的 **細噪點米紙紋理**（PNG，2x retina），平鋪
- 透明度 30%，疊在 `--paper-bg` 上
- 角落微邊緣陰影（內陰影 inset），讓畫面像在「攤開的舊紙」上

### 5.6 互動回饋
- Hover 節點：節點放大 1.15x，墨色描邊加深
- 選中節點：金色描邊（同突破點色）+ 連結邊變高亮
- 摺疊節點：點兩下父節點，子節點以「縮回原點」動畫消失（300ms ease-out），父節點顯示 `+N` 角標

### A. TopBar（頂部，固定）

- 每層節點顯示：類別圖示 + 標題 + 子節點數
- 點父節點：圖譜上同步摺疊/展開
- 點葉節點：圖譜上 zoom 到該節點 + 開 InfoCard
- 收起按鈕：左下角「›」，按下整個 sidebar 滑出，圖區放大

### C. 主圖區（中央，Canvas）
- 力導向圖預設視圖
- 階層用 d3-hierarchy 預先計算固定半徑同心圓位置（最頂層在中心、第 2 層中環、第 3 層外環）
- 跨越邊在這個骨架上「飄」
- 滾輪 zoom、左鍵拖移畫布、左鍵點節點選中、雙擊父節點摺疊
- 沒選中任何節點時，hover 顯示節點標題 tooltip
- 選中節點：相關節點高亮、其餘灰化（30% 透明）

### D. Minimap（右下角浮動）
- 160×120px 半透明卡片
- 顯示完整圖佈局縮圖 + 當前視窗範圍方框
- 拖動方框可平移主圖
- 點任意位置直接跳過去
- 收合按鈕：右上「‒」

### E. InfoCard（右側 slide-in）
- 預設隱藏；選中節點時從右側 350ms 滑入
- 內容區（從上到下）：
  1. 節點標題（思源宋體 28px）+ 類別 chip + 重要性星等
  2. 年份：`1850 – 1923`（不確定加問號）
  3. 摘要（summary）
  4. 主圖（如有）
  5. 詳細介紹（long_text，markdown 渲染）
  6. **突破點區**（如有 breakthrough_note）：金色邊框 + 「★ 突破點」標題，內容以小字斜體
  7. 媒體區：圖片 lightbox 縮圖、YouTube embed、音檔 player
  8. 相關節點：依 edge 類別分組顯示，點擊跳轉
  9. 出處（sources）：下方淺灰小字
  10. 永久連結：點擊複製當前 URL（含 ?node= 參數）
- 關閉按鈕：右上「✕」；按 Esc 也關

### F. BottomBar（底部，固定）
- **左**：時間軸滑桿（雙端點）
  - 範圍 1850–2025（依資料動態計算）
  - 拖動立即過濾
  - 滑桿下方有資料密度長條圖（每 10 年的節點數，灰色直條）
- **中**：邊類切換 segmented：`[全部][階層][跨越][時序]`
- **右**：圖例（7 大類色塊，可點即篩）+ 重置按鈕

## 7. 階層 + 跨越混合關係的處理細則

### 7.1 雙佈局策略：階層骨架 + 跨越彈性
1. 第一階段（page load）：用 `d3-hierarchy` 計算純樹狀佈局
   - 第 1 層 7 個節點放在中心半徑 150px 圓周上
   - 第 2 層 (~30 個) 放在半徑 350px 圓周上，按父節點扇形分組
   - 第 3 層 (100–900 個) 放在半徑 600+ px 同心環，按祖父分組
2. 第二階段：把這些位置作為力模擬的初始值，且階層邊給高 link strength（`forceLink().strength(0.8)`）
3. 第三階段：跨越邊以低 strength 加入（`0.1`），讓它們「飄著」連，不破壞主結構
4. 跨越邊渲染為貝茲曲線（避免直線穿過其他節點）

### 7.2 摺疊行為
- 預設：第 1、2 層全展開；第 3 層摺疊（顯示父節點上 `+N` 角標）
- 摺疊狀態存在 URL `?collapsed=cat1,subcat3,subcat7`
- 摺疊時：
  - 子節點動畫飛回父節點位置 → 消失
  - 涉及子節點的跨越邊改連到父節點（虛線變短虛線，標註「+3」表示底下還有 3 條）
- 一鍵「全展開」/「全摺疊」按鈕在工具列

### 7.3 三檔位邊類切換
- **全部**：階層邊（粗實）+ 跨越邊（細虛）+ 時序邊（朱點線）同時顯示
- **只看階層**：只顯示樹結構，最乾淨，適合理解「整體分類框架」
- **只看跨越**：把樹結構淡化（節點留著但邊全隱），只顯跨越邊，適合看「人事物的橫向關係」
- 切換有 200ms 淡入淡出過渡，不重新跑 force

## 9. 篩選與時間軸機制

### 9.1 篩選管線
篩選依序套用，結果透過 `useMemo` 快取：

1. **時間範圍**：節點 year/year_end 與滑桿區間有交集
   - 沒有 year 的節點視為「永恆」，永遠通過（沿用 Zhudong 邏輯）
   - 有 year 但 year_uncertain=true 的節點通過時加問號標記
2. **類別**：節點 category 在已勾選清單中
3. **重要性**：importance >= 滑桿值（預設 1，全通過）
4. **突破點 only**：toggle 開啟時，breakthrough_note 不為空才通過
5. **搜尋**：搜尋詞在 title/summary/long_text/tags 出現
6. **摺疊**：被摺疊父節點的子節點不渲染（但仍計入相關性計算）

→ 邊的篩選：source 和 target 都通過才保留

### 9.2 時間軸視圖（替代視圖）
- 選 BottomBar 上方「時間軸」toggle 切換
- 橫軸：年份（連續軸，無分期色帶 — 你要的）
- 縱軸：類別泳道（7 條）
- 節點：點事件以圓點、跨年事件以橫條
- 邊：預設隱藏，選中節點時才顯示其相關邊（避免時間軸被線條淹沒）
- Zoom：橫向縮放年份軸（10 年 → 1 年細節）

## 10. 搜尋與 InfoCard 細節

### 10.1 搜尋
- 即時搜尋：輸入時 debounce 150ms，從 `title`、`summary`、`long_text`、`tags` 模糊匹配
- 結果分組：人物 / 部落 / 事件 / 文獻
- 每結果顯示：標題 + 類別 chip + 年份 + 摘要前 30 字
- 鍵盤：上下選、Enter 跳轉、Esc 關
- 點結果：圖譜飛行聚焦 + 開 InfoCard + URL 同步

### 10.2 InfoCard 媒體處理
- 圖片：縮圖 grid（最多 4 張），點擊開全屏 lightbox（左右鍵切換、Esc 關）
- YouTube：用 iframe embed，`?rel=0` 避免播完跳到亂入內容
- 音檔：HTML5 `<audio>` + 自製簡單播放控制
- 所有媒體 lazy load（IntersectionObserver）

### 10.3 相關節點清單
- 依 edge_type 分組：
  - 階層：父節點、子節點（顯示樹狀）
  - 跨越：以 relation_label 群組
  - 時序：依時間排序
- 點任一相關節點：跳轉並更新 InfoCard，主圖不重置（保持上下文）

## 11. URL 狀態同步協議

URL 是單一真實來源，所有狀態都可分享。

| 參數 | 範例 | 說明 |
|---|---|---|
| `node` | `?node=person_莎韻` | 當前選中節點 |
| `view` | `&view=timeline` | 視圖（graph/timeline，預設 graph） |
| `cat` | `&cat=tribe,person,event` | 啟用的類別（預設全部） |
| `years` | `&years=1900-1945` | 時間範圍 |
| `edges` | `&edges=hierarchy` | 邊類顯示（all/hierarchy/cross/temporal，預設 all） |
| `imp` | `&imp=3` | 最低重要性（預設 1） |
| `bt` | `&bt=1` | 只看突破點（預設 0） |
| `q` | `&q=武塔` | 搜尋詞 |
| `collapsed` | `&collapsed=cat_culture` | 摺疊的父節點 id（逗號分隔） |

實作：自製 `useUrlState` hook，雙向同步。`replaceState` 用於連續操作（拖滑桿），`pushState` 用於離散操作（點節點、開時間軸視圖），讓上一頁/下一頁有意義。

## 12. 資料管線（建議方案 B 細節）

### 12.1 Excel 範本
給文史工作者一個 `template.xlsx`，內含 3 個分頁：
- `nodes`：欄位如 §4.1
- `edges`：欄位如 §4.2
- `說明`：每個欄位的填法、enum 對照表、範例資料

第一行用顏色標頭（必填欄黃底、選填欄白底），第二行是說明，第三行起是資料。

### 12.2 GitHub Action 流程
觸發條件：`data/source.xlsx` 變更
步驟：
1. checkout
2. setup-python 3.11
3. pip install openpyxl jsonschema
4. 跑 `scripts/excel_to_json.py`：
   - 讀 Excel
   - 跑驗證（id 唯一、parent 存在、edge 端點存在等）
   - 驗證失敗 → 報錯 + 上傳 `validation_report.html` 為 artifact
   - 驗證成功 → 寫 `data/data.json`
5. 如果 JSON 有變化，commit 回主分支（用 `peter-evans/create-pull-request` 或直接 push）

### 12.3 驗證報告格式
HTML 表格：每個錯誤一行，欄位：類型 / 位置（sheet 名 + 儲存格） / 訊息 / 建議修復
這比 Apps Script 在 Sheet 裡塗顏色更難看到，所以也同步在 PR description 自動貼錯誤摘要。

## 13. 效能策略（撐到 1000 節點）

| 項目 | 策略 |
|---|---|
| 節點繪製 | Canvas，requestAnimationFrame 統一繪製 |
| 命中測試 | d3-quadtree，每 tick 重建 |
| Label 渲染 | 只渲染當前視窗 + zoom > 0.8 時的節點 label（用 viewport culling） |
| Force 模擬 | 初始 alpha=1，穩定後（alpha < 0.01）暫停 simulation tick |
| 圖片 | 縮圖預生成 200x200，原圖 lazy load |
| JSON | 開 gzip（GitHub Pages 內建） + 用 commit hash 在檔名做版本控制 |
| 篩選 | useMemo 快取，依賴最小化 |
| 摺疊 | 摺疊的子節點不進入 force simulation（從 nodes 陣列移除，而不只是隱藏） |

效能目標：
- 1000 節點初始載入 < 2 秒
- 拖動畫布 60fps
- 切換篩選 < 200ms

## 14. 行動裝置體驗

純桌面為主，但 RWD 至少不破版。

| 螢幕寬 | 行為 |
|---|---|
| ≥ 1024px | 完整版面（左 sidebar + 中圖 + 右 InfoCard） |

- 單指拖移 = 平移

- 長按 = 摺疊/展開（取代雙擊）

## 18. 風險與待你確認

### 18.1 已知風險
1. **泰雅族口傳資料的時間性**：很多敘事沒有明確年份，year_uncertain 機制夠不夠？是否要做「無時間泳道」？
2. **族語呈現**：title 是用「Pyahaw」還是「碧候」還是兩者？建議 schema 加 `title_alt` 欄位存族語名
3. **倫理審查**：部落會議是否同意公開？某些口傳是否有性別/身份限制？要在資料管線加「公開等級」欄位（`public/restricted/private`）
4. **規模膨脹**：1000 節點是上限，超過要分期或做地圖切割
5. **字體授權**：思源系列為 SIL OFL 可商用、可自託管，無問題；EB Garamond 同樣 OFL

### 18.2 需要你回覆的事
1. **南澳資料表的存取權**：目前我這邊抓不到那份 Google Sheets。請開放「任何取得連結者皆可讀」，或把欄位 header + 5 筆樣本貼過來，我才能依你的真實資料微調 schema（v2 規劃）
2. **是否做泰雅族語並列**：影響 schema 是否加 `title_alt`、`category_alt`
3. **倫理公開等級欄位是否需要**：影響資料管線與 UI（要不要顯示「此節點需登入查看」之類）
4. **是否與既有平台連動**（如台灣記憶、原民會數位典藏）：影響節點要不要有 `external_link` 欄位
5. **網域**：要用 GitHub Pages 子網域還是自訂網域？

- **v2 規劃**：依真實資料對齊 schema、給出具體的 50 筆種子資料範例

- **Excel 範本**：直接生成 template.xlsx 給文史工作者用
